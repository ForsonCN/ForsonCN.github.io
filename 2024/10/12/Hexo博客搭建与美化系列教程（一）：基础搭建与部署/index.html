Hexo博客搭建与美化系列教程（一）：基础搭建与部署

John Doe2024-10-122024-10-12

一、环境准备

  1. 安装Node.js

• Node.js是运行Hexo的基础环境。访问 Node.js官网，根据你的操作系统下载对应的安装程序。

• 安装完成后，打开命令行工具（Windows用户可按 Win + R，输入 cmd 回车打开命令提示符），输入以下命令检查是否安装成功：
node -v
2. 安装Git

• Git用于版本控制和部署博客。进入 Git官网，下载适合你系统的版本。

• 安装过程中可使用默认设置。安装完成后，同样在命令行中输入以下命令检查安装情况：
git –version
3. 配置Git和GitHub

• 配置用户名和邮箱
在命令行中输入以下命令，将 <你的用户名> 和 <你的邮箱> 替换为实际信息：
git config –global user.name “你的用户名”
git config –global user.email “你的邮箱”
• 生成SSH密钥并配置到GitHub

  1. 生成SSH密钥对，在命令行中输入以下命令，将 <你的邮箱> 替换为你注册GitHub使用的邮箱：
    ssh-keygen -t rsa -C “你的邮箱”
    生成过程中一路回车使用默认设置即可。

  2. 找到生成的公钥文件。在Windows系统中，公钥文件通常位于 C:\Users\你的用户名.ssh\id_rsa.pub。用记事本打开该文件，复制其中的内容。

  3. 登录GitHub，点击右上角头像，选择 Settings，再选择 SSH and GPG keys，点击 New SSH key，在 Title 处可随意填写一个名称，将复制的公钥内容粘贴到 Key 文本框中，然后点击 Add SSH key。

  4. 测试连接是否成功，在命令行中输入：
    ssh -T git@github.com
    首次连接会提示是否继续，输入 yes。如果看到类似 Hi 你的用户名! You’ve successfully authenticated, but GitHub does not provide shell access. 的信息，则表示连接成功。

  5. 创建GitHub仓库

• 登录GitHub，点击右上角的 + 按钮，选择 New repository。

• 仓库名称必须为 <你的用户名>.github.io（注意前缀必须为你的用户名），可见性选择 Public，然后点击 Create repository。

二、Hexo安装与初始化

  1. 安装Hexo

• 打开命令行工具，输入以下命令安装Hexo：
npm install -g hexo-cli
2. 初始化Hexo项目

• 创建一个新的文件夹用于存放博客项目（例如 D:\HexoBlog），在该文件夹内右键点击，选择 Open Git Bash here（如果使用的是Windows系统且安装了Git）。

• 在打开的 Git Bash 中输入以下命令初始化Hexo项目：
hexo init blog-demo
cd blog-demo
npm install
3. 本地运行Hexo

• 在命令行中输入以下命令启动本地服务器：
hexo s
• 打开浏览器，访问 http://localhost:4000，即可看到初始化后的Hexo博客页面。

三、Hexo部署到GitHub Pages

  1. 安装部署插件

• 在命令行中输入以下命令安装 hexo-deployer-git 插件：
npm install hexo-deployer-git –save
2. 配置部署信息

• 打开博客项目根目录下的 _config.yml 文件，找到 deploy 部分，进行如下配置（将 <你的用户名> 替换为实际的GitHub用户名）：
deploy:
type: git
repository: git@github.com:<你的用户名>/<你的用户名>.github.io.git
branch: main
3. 部署到GitHub Pages

• 在命令行中输入以下命令进行部署（可缩写为 hexo cl; hexo g; hexo d）：
hexo clean && hexo generate && hexo deploy
• 部署过程中可能需要输入GitHub的用户名和密码。如果看到 Deploy done 的提示，则表示部署成功。

• 稍等片刻后，在浏览器中访问 https://<你的用户名>.github.io，即可看到部署后的博客页面。

Hexo博客搭建与美化系列教程（二）：主题美化与功能扩展

一、主题选择与安装

  1. 主题选择

• Hexo有众多主题可供选择，可根据个人喜好和需求挑选。本教程以安知鱼主题为例，其项目地址为 https://github.com/anzhiyu-c/hexo-theme-anzhiyu，官方文档在 https://docs.anheyu.com/initall.html。

  1. 主题安装

• 在博客根目录（即包含 _config.yml 文件的目录）下，打开命令行工具，输入以下命令安装主题：
git clone -b main https://github.com/anzhiyu-c/hexo-theme-anzhiyu.git themes/anzhiyu
• 由于该主题使用了 pug 和 stylus，首次安装需要安装对应的渲染器，在命令行中输入以下命令：
npm install hexo-renderer-pug hexo-renderer-stylus –save
二、主题配置

  1. 应用主题

• 打开 _config.yml 文件，将 theme 的值修改为 anzhiyu：
theme: anzhiyu
2. 覆盖配置

• 为了避免在主题更新时丢失自定义配置，需要进行覆盖配置操作。

• 如果是Linux系统，在命令行中输入以下命令：
cp -rf./themes/anzhiyu/_config.yml./_config.anzhiyu.yml
• 如果是Windows系统，则手动将 themes/anzhiyu/_config.yml 文件复制到根目录下，并将其重命名为 _config.anzhiyu.yml。

• 以后如果修改主题配置，都只需修改 _config.anzhiyu.yml 文件即可。需要注意的是，只要存在于 _config.anzhiyu.yml 的配置都是高优先级，修改原 _config.yml 中的主题相关配置是无效的。每次主题更新可能存在配置变更，请注意更新说明，可能需要手动对 _config.anzhiyu.yml 进行同步修改。可以通过 hexo g –debug 查看覆盖配置是否生效。

三、生成标签页和分类页

  1. 生成标签页

• 在命令行中输入以下命令生成标签页：
hexo new page tags
• 找到 source/tags/index.md 文件，打开并修改添加 type: “tags”，如下所示：

title: 标签
date: 2024-07-05 03:36:02
type: “tags”
comments: false
top_img: false

  1. 生成分类页

• 在命令行中输入以下命令生成分类页：
hexo new page categories
• 找到 source/categories/index.md 文件，打开并修改添加 type: “categories”，如下所示：

title: 分类
date: 2024-07-05 03:36:48
aside: false
top_img: false
type: “categories”

一、Hexo博客搭建基础

（一）环境准备的重要性及原理

  • Node.js和Git的作用
    • Node.js是一个基于Chrome V8引擎的JavaScript运行时环境，Hexo是基于Node.js开发的，它依赖Node.js提供的运行环境来执行各种脚本和任务，如解析主题模板、生成静态页面等。
    • Git是一个分布式版本控制系统，用于管理博客的代码版本和进行部署。在开发过程中，我们可以使用Git来记录对博客的每一次修改，方便回溯和协作。同时，通过配置Git与GitHub，我们可以将博客代码推送到远程仓库进行保存和部署。
  • 正确配置Git和GitHub的关键步骤及原因
    • 配置用户名和邮箱：这一步是为了在使用Git时能够正确标识提交代码的作者信息。当多人协作开发博客或者查看代码历史记录时，能够清楚地知道每一次提交是谁做出的。
    • 生成SSH密钥并配置到GitHub：SSH（Secure Shell）是一种网络协议，用于在不安全的网络上提供安全的远程登录和其他安全网络服务。通过生成SSH密钥对并将公钥配置到GitHub，我们可以在本地计算机和GitHub服务器之间建立安全的连接，无需每次推送代码时都输入用户名和密码，提高了操作的便利性和安全性。

（二）Hexo安装与初始化的详细过程及注意事项

  • 安装Hexo的原理及不同系统可能出现的问题
    • Hexo的安装过程实际上是通过Node.js的包管理器npm从互联网上下载Hexo的相关模块和依赖，并将它们安装到全局环境中。在不同的操作系统上，可能会出现一些与权限相关的问题。例如，在Windows系统中，如果没有以管理员身份运行命令行工具，可能会因为权限不足而无法将Hexo正确安装到全局环境中。
    • 解决方法是在安装Hexo时确保以管理员身份运行命令行工具（右键点击命令行工具的快捷方式，选择“以管理员身份运行”）。
  • 初始化Hexo项目的具体操作及文件结构介绍
    • 初始化Hexo项目时，hexo init命令会在指定的文件夹下创建一个基本的Hexo项目结构。其中：
      • node_modules文件夹包含了Hexo运行所需的所有依赖模块。
      • scaffolds文件夹存放着用于生成不同类型文章的模板文件，如post.md用于生成普通文章，page.md用于生成页面（如标签页、分类页等）。
      • source文件夹是用于存放用户撰写的文章和其他资源文件（如图片、样式表等）的地方，文章通常以.md格式存放在_posts子文件夹下。
      • themes文件夹用于存放Hexo的主题文件，用户可以通过更换不同的主题来改变博客的外观。
      • _config.yml是Hexo项目的主配置文件，用于配置博客的各种参数，如网站标题、作者信息、主题设置、部署设置等。
      • .npmignore文件用于指定在发布项目时哪些文件不需要被包含在内。

（三）本地运行Hexo的原理及如何查看效果

  • 本地服务器启动原理
    • 当输入hexo s命令时，Hexo会启动一个本地的Web服务器，这个服务器会监听在本地的4000端口（默认端口）。它会根据用户配置的主题和文章内容，动态地生成相应的HTML页面，并提供给浏览器访问。
  • 如何通过浏览器查看本地博客效果及可能出现的问题
    • 打开浏览器，输入http://localhost:4000，浏览器会向本地服务器发送请求，服务器会将生成的HTML页面返回给浏览器进行显示。
    • 可能出现的问题包括：
      • 端口被占用：如果本地已经有其他程序占用了4000端口，会导致服务器无法正常启动。解决方法是查看系统进程，找到占用该端口的程序并关闭它，或者修改Hexo的配置文件，将服务器监听的端口改为其他未被占用的端口（在_config.yml文件中找到server部分，修改port参数的值）。
      • 页面显示异常：可能是由于主题配置错误或者文章内容格式问题导致的。可以检查主题配置文件和文章的.md文件，确保格式正确，内容完整。

二、Hexo博客部署到GitHub Pages

（一）部署插件的作用及安装原理

  • hexo-deployer-git插件的作用
    • hexo-deployer-git插件是Hexo用于将生成的静态页面部署到远程Git仓库（如GitHub Pages）的工具。它会将本地生成的静态文件（通常存放在public文件夹下）按照配置的仓库地址和分支信息，推送到远程仓库中，使得博客能够在互联网上访问。
  • 安装插件的原理及可能出现的问题
    • 安装插件是通过npm命令从互联网上下载插件的相关模块和依赖，并将它们安装到项目的本地环境中。可能出现的问题包括：
      • 网络问题：如果网络不稳定或者无法访问插件的官方仓库，可能会导致安装失败。解决方法是检查网络连接，尝试更换网络环境或者使用代理服务器。
      • 版本冲突：如果项目中已经存在与插件相关的模块，并且版本不兼容，可能会导致安装失败或者插件无法正常工作。解决方法是查看插件的文档，了解其兼容的版本范围，或者尝试升级或降级相关模块的版本。

（二）部署信息配置的详细解释及注意事项

  • repository和branch参数的含义及如何正确填写
    • repository参数用于指定远程Git仓库的地址，格式为git@github.com:<用户名>/<仓库名>.git。其中<用户名>是你的GitHub用户名，<仓库名>是你创建的用于存放博客代码的仓库名称（通常为<用户名>.github.io）。
    • branch参数用于指定将文件推送到远程仓库的哪个分支，通常为main（GitHub默认的主分支名称，以前可能是master）。
    • 在填写这些参数时，要确保信息准确无误，否则会导致部署失败。可以通过查看GitHub仓库的设置页面来获取正确的仓库地址和分支信息。
  • 其他可能影响部署的配置项及注意事项
    • 在_config.yml文件中，还有一些其他的配置项可能会影响部署效果，如deploy部分的message参数（用于指定每次部署时的提交信息）。如果不设置该参数，默认会使用Hexo deploy作为提交信息。在实际应用中，可以根据需要设置一个有意义的提交信息，以便更好地跟踪部署历史。

（三）部署过程及常见问题解决

  • 部署过程的详细步骤及命令执行原理
    • 部署过程主要通过hexo clean、hexo generate和hexo deploy三个命令来完成。
      • hexo clean命令会删除之前生成的静态文件和缓存，确保每次部署都是基于最新的文章内容和配置。
      • hexo generate命令会根据当前的文章内容和配置，重新生成静态页面，并将生成的文件存放在public文件夹下。
      • hexo deploy命令会根据配置的部署信息，将public文件夹下的文件推送到远程Git仓库中。
    • 这些命令的执行原理是基于Hexo的内部机制和插件的功能实现的。通过执行这些命令，Hexo能够有条不紊地完成博客的部署工作。
  • 常见问题及解决方法
    • 部署失败的可能原因及解决方法
      • 配置错误：如前面提到的repository和branch参数填写错误，或者其他配置项设置不当。解决方法是仔细检查_config.yml文件中的部署配置部分，确保信息准确无误。
      • 权限问题：如果没有足够的权限访问远程Git仓库，会导致部署失败。解决方法是检查GitHub仓库的权限设置，确保你拥有推送文件的权限。
      • 网络问题：如果网络不稳定或者无法访问远程Git仓库，会导致部署失败。解决方法是检查网络连接，尝试更换网络环境或者使用代理服务器。
      • 插件问题：如果hexo-deployer-git插件出现故障或者版本不兼容，会导致部署失败。解决方法是重新安装插件，或者尝试升级或降级插件的版本。
    • 如何判断部署是否成功及后续操作
      • 判断部署是否成功可以通过查看命令行输出，如果看到Deploy done的提示，则表示部署成功。
      • 部署成功后，可以在浏览器中访问https://<用户名>.github.io，查看博客是否能够正常访问。如果无法访问，可以检查域名解析是否正确，或者查看GitHub Pages的设置是否正确。
